## 一、产品定位

### 1.1 是什么

**Rho = Research + ρ (多Agent协作)**

一个**AI投研助手**，输入股票代码，自动扒财报、新闻、技术指标，生成专业投资研究报告。

**不是这样的：**
- ❌ 不是交易机器人（不下单）
- ❌ 不是股票筛选器（不给排名）
- ❌ 不是数据看板（不只是展示数据）

**是这样的：**
- ✅ 是**投资研究伴侣**——帮用户理解一家公司
- ✅ 是**教育工具**——让普通人看懂投资逻辑
- ✅ 是**决策辅助**——提供多维度分析和风险评估

### 1.2 目标用户

| 用户类型 | 需求 | Rho如何满足 |
|---------|------|------------|
| 股市新手 | 想投资但不知从何分析 | 一键生成研报，看懂一家公司 |
| 忙碌职场人 | 想投资但没时间深度研究 | 5分钟了解关键信息 |
| 价值投资者 | 需要系统化的基本面分析 | 多Agent协作的深度研究 |
| 学生/小白 | 学习投资分析思路 | 流式输出展示分析逻辑 |

### 1.3 解决什么问题

**用户痛点：**
1. "我不知道怎么分析一家公司" → Rho提供完整的分析框架
2. "研报都是收费的" → Rho免费生成投资简报
3. "看了数据不知道该买不该买" → Rho给出置信度和风险评估
4. "AI生成的内容信不过" → 流式输出让用户看到分析过程

## 二、核心竞争力（Competitive Edge）

### 2.1 核心差异化

| 维度 | 其他产品 | Rho |
|-----|---------|-----|
| **研究深度** | 单维度数据罗列 | 4个分析师多维并行 + 综合 + 风险评估 |
| **过程透明** | 黑盒等结果 | 流式输出，实时看到每个Agent的分析 |
| **风险可见** | 只给结论 | 明确的风险评分和风险因素 |
| **国内市场** | 照搬国外 | A股特色：财报规则、公告解读 |

### 2.2 杀手锏功能

#### ① 流式输出（Streaming Progress）
- 用户输入股票 → 立即看到"基本面分析中..." → "情绪分析中..." → 实时进度
- **为什么是杀手锏**：让用户"参与"研究过程，不是黑盒等结果，增强信任感

#### ② 多Agent协作研究
- Supervisor（调度）→ 4个 Analyst（并行）→ Synthesizer（综合）→ RiskEvaluator（风险）→ ReportGenerator（报告）
- **为什么是杀手锏**：模拟专业投研团队分工，而不是单一AI问答

#### ③ 风险评估
- 明确的风险等级（R1-R5）和风险因素列表
- **为什么是杀手锏**：不只告诉"买"，还告诉"为什么"和"需要注意什么"

### 2.3 vs 参考项目

| 参考项目 | 核心定位 | 我们的差异 |
|---------|---------|-----------|
| **TradingAgents** | 量化交易 | 我们做投研教育，不做交易决策 |
| **gpt-researcher** | 通用研究 | 我们专注金融投研，有深度行业知识 |
| **同花顺** | 数据工具 | 我们做分析，数据用户自己看 |
| **东方财富** | 研报平台 | 我们用AI生成，不是人工撰写 |

**面试回答：**
> "Rho和TradingAgents的区别？他们做量化交易，我们做投教研究。他们输出交易信号，我们输出投资建议。目标用户不同，我们服务的是想学习投资但不知从何入手的人。"

## 三、功能架构

### 3.1 核心功能矩阵

| 功能 | 描述 | 优先级 | 状态 |
|-----|------|-------|-----|
| **投资研究** | 输入股票→生成研究报告 | P0 | ✅ 完成 |
| **流式输出** | 实时显示Agent进度 | P0 | ✅ 完成 |
| **K线图表** | 技术分析可视化 | P0 | ✅ 完成 |
| **股票对比** | 多股票横向对比 | P1 | ✅ 完成 |
| **监控告警** | 价格/RSI/成交量告警 | P1 | ✅ 完成 |
| **组合管理** | 仓位建议+风险分析 | P1 | ✅ 完成 |
| **策略回测** | MA/RSI/动量策略验证 | P2 | ✅ 完成 |
| **报告导出** | PDF/HTML导出 | P2 | ✅ 完成 |

### 3.2 用户流程

```
┌─────────────────────────────────────────────────────────────┐
│ 用户旅程                                                            │
└─────────────────────────────────────────────────────────────┘

① 发现阶段
   用户听说Rho → 打开首页 → 看到功能介绍 → "这个好像有用"

② 研究阶段
   输入股票代码 → 看到流式进度 → 等待报告生成 → 阅读研究报告

③ 决策阶段
   查看风险评估 → 参考投资建议 → 决定是否投资

④ 跟踪阶段
   设置监控告警 → 组合管理 → 定期回顾
```

```
首页 (/)
├── 研究 (/research)     ← 核心入口，股票研究
├── 对比 (/compare)      ← 多股票对比

├── 组合 (/portfolio)    ← 仓位建议
├── 回测 (/backtest)     ← 策略验证
└── 导出 (/exports)      ← 报告导出
```

## 四、数据设计

### 4.1 数据来源

| 数据类型 | 来源 | 备注 |
|---------|-----|------|
| 股票价格 | yfinance | 免费，真实 |
| 财务数据 | 东方财富API | A股特色 |
| 新闻资讯 | Tavily搜索 | 实时 |
| 技术指标 | 本地计算 | RSI/MACD/KDJ |

### 4.2 数据捏造原则

**可以捏造的场景：**
- 演示时没有真实API key
- 测试边界情况（如股票代码无效）
- UI原型阶段

**捏造的数据必须有真实感：**
```json
// ❌ 坏数据
{"price": 100, "change": 5}

// ✅ 好数据
{"price": 1680.50, "change": 23.45, "changePercent": 1.42, "volume": 3245678}
```

## 五、用户体验

### 5.1 核心体验原则

1. **不要让用户等待焦虑**
   - 流式输出让用户看到进度
   - 骨架屏/加载动画

2. **不要让用户看不懂**
   - 专业技术术语要有中文解释
   - 风险评估要直观（如"中高风险 R3"）

3. **不要让用户迷茫**
   - 每个页面有明确的目标
   - 操作路径清晰

### 5.2 交互细节

| 场景 | 体验要求 |
|-----|---------|
| 输入股票代码 | 自动格式化，验证有效性 |
| 等待研究 | 流式进度 + Agent状态展示 |
| 查看报告 | 分模块折叠，可展开详情 |
| 查看图表 | 支持缩放、十字光标 |
| 添加告警 | 即时反馈，成功/失败提示 |

### 5.3 动画规范

**必须使用Framer Motion：**
- 页面切换：slide + fade
- 卡片加载：stagger animation
- 数据更新：morphing transition
- 错误提示：shake + red highlight

**禁止：**
- ❌ 纯静态页面
- ❌ 无过渡的即时切换
- ❌ 生硬的AI蓝紫色

## 六、技术规范

### 6.1 技术栈

| 层级 | 技术 | 选型原因 |
|-----|-----|---------|
| 前端框架 | Next.js 15 | App Router + SSR |
| UI组件 | shadcn/ui | 成熟、可定制 |
| 动画 | Framer Motion | 声明式动画 |
| 图表 | lightweight-charts v5 | TradingView，专业K线 |
| 后端框架 | Flask | 轻量、灵活 |
| Agent编排 | LangGraph | 多Agent状态管理 |
| LLM | MiniMax M2 | 成本低、效果好 |

### 6.2 API规范

**Base URL:** `http://localhost:8001`

| 端点 | 方法 | 描述 |
|-----|------|------|
| `/api/research` | POST | 批量研究股票 |
| `/api/research/stream` | GET | SSE流式研究 |
| `/api/stock/chart/<code>` | GET | K线图表数据 |
| `/api/compare` | POST | 股票对比 |
| `/api/compare/rank` | POST | 股票排名 |
| `/api/monitor/*` | * | 监控告警API |
| `/api/portfolio/*` | * | 组合管理API |
| `/api/backtest/*` | * | 回测API |
| `/api/export/*` | * | 导出API |

## 八、竞品分析

### 8.1 直接竞品

| 产品 | 优点 | 缺点 | Rho机会 |
|-----|------|------|--------|
| 同花顺 | 数据全 | 门槛高，要用户自己分析 | 降低分析门槛 |
| 东方财富 | 研报专业 | 收费 | 免费+AI |
| TradingAgents | Agent架构 | 面向交易，非研究 | 专注投研教育 |

### 8.2 差异化定位

**Rho的独特位置：**
- 比同花顺/东财**更简单**——不用学看盘
- 比通用AI**更专业**——懂金融投资
- 比研报平台**更便宜**——免费
